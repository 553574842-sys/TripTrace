# TripTrace Security Policy

## Reporting a vulnerability

Please report security issues privately to the current TripTrace maintainer.
Do not include API keys, passwords, personal travel data, database files, or
uploaded documents in a public issue.

Include the affected version, reproduction steps, expected impact, and any
suggested mitigation. Security reports will be acknowledged and prioritized
before public disclosure.

## Supported deployment

This policy covers the TripTrace application built from this repository.
Production instances should use HTTPS, a strong `ENCRYPTION_KEY`, restricted
file permissions, and regular off-site backups.
