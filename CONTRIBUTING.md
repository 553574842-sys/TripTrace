# Contributing to TripTrace

TripTrace focuses on a Chinese-first domestic travel-planning experience.

Before submitting a change:

1. Open an issue describing the problem or proposed feature.
2. Keep changes scoped and compatible with existing data.
3. Add or update tests for changed behavior.
4. Run the relevant client and server tests.
5. Do not commit `.env` files, API keys, databases, logs, or user uploads.

Pull requests should explain the user-facing impact, verification performed,
and any deployment or migration steps.
